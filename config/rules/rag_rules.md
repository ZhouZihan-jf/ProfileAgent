# RAG API 调用规则

## 何时调用 RAG API

### 必须调用的情况
1. **画像模板中有 source=rag 的字段时**，必须调用 RAG 获取相关知识
2. **套餐信息中缺乏足够的市场上下文**（如竞品信息、行业趋势）时调用
3. **需要行业标准术语或分类**时调用，以确保画像专业性

### 不调用的情况
1. 画像模板中无 source=rag 字段 → 跳过
2. TMF 输入中已包含足够的上下文信息 → 可以跳过
3. RAG API 未配置或不可用 → 必须跳过

## 查询生成规则

1. 为每个 source=rag 的字段生成 1-3 个查询
2. 查询应包含：套餐名称关键字 + 字段描述中的关键词
3. 查询可以是中文或英文，建议双语
4. 每个查询不超过 100 个字符

## 检索结果使用规则

1. 优先使用 score > 0.7 的文档
2. 如无高分文档，使用 Top-3 结果并在画像中标注"知识库置信度较低"
3. 检索结果与 TMF 输入冲突时以 TMF 为准

## 示例

### 场景：5G 大流量套餐
- 字段: competitive_advantages
- 查询: ["5G大流量套餐 竞品分析", "5G unlimited data plan competitor comparison"]
- 预期结果: 其他运营商的类似套餐对比数据
